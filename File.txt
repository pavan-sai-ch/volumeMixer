Objective: Implement the per-app volume control logic using CoreAudio and the CATap API.
Instructions:
1. Model: Create Models/AudioProcess.swift. It must include PID, app name, app icon (using NSWorkspace), and a thread-safe volume Double (0.0 to 1.0).
2. Core Service: Create Core/AudioManager.swift. Implement a polling mechanism to find apps where kAudioHardwarePropertyProcessIsAudible is true.
3. The Tap: For each audible PID, implement a CATapDescription. Use AudioObjectSetPropertyData to map the volume property of our model to the kAudioDevicePropertyVolumeScalar of the tap.
4. UI Integration: Create UI/MenuBarView.swift using MenuBarExtra. Iterate through the active processes in AudioManager and display them with a Slider and their high-res App Icon.
Constraints: Ensure all Core Audio C-pointers are properly released in deinit to prevent memory leaks. Use @MainActor for UI updates.

A Note on the "Icon"
When Claude asks for the icons, remind it to use NSWorkspace.shared.icon(forFile:) rather than searching for specific image assets. This ensures that every app on your MacBook—whether it's Chrome, Slack, or a game—shows its native icon in your volumeMixer.
